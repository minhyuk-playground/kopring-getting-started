# 게시판

---
게시판을 구현한다.

## 비즈니스 규칙(요구사항)

---

### 게시물

- 게시물은 반드시 1글자 이상의 제목을 가지고 있어야 한다.
- 게시물의 제목이 빈문자열이라면 게시물을 저장할 수 없다.
- 게시물의 본문 내용은 비어있을 수 있다.
- 시용자는 게시물을 발행할 수 있다.
- 사용자는 게시물을 수정할 수 있다.
    - 작성자 본인만 게시물을 수정할 수 있다.
    - 삭제된 게시물은 수정할 수 없다.
- 작성자는 게시물을 삭제할 수 있다.
    - 작성자 본인만 게시물을 삭제할 수 있다.
    - 이미 삭제된 게시물은 삭제할 수 없다.
- 모든 사용자는 게시물 목록을 조회할 수 있다.
    - 발행이 완료된 게시물만 목록에 노출되어야 한다.
    - 임시 저장된 게시물 및 삭제된 게시물은 게시물 목록에 노출되지 않아야 한다.
    - 제목으로 검색이 가능해야한다.
- 모든 사용자는 게시물의 상세 내용을 조회할 수 있다.

### 초안

- 사용자는 신규 게시물의 초안을 저장할 수 있다.
- 사용자는 초안을 발행할 수 있다.
    - 작성자 본인만 초안을 수정할 수 있다.
    - 초안은 발행되어지면 삭제되어야 한다.
    - 발행이 되어지면 게시물로 등록되어야 한다.
    - 초안의 제목은 1글자 이상이어야 한다.
    - 초안의 내용은 비어있을 수 있다.
- 사용자는 초안을 수정할 수 있다.
    - 작성자 본인만 초안을 수정할 수 있다.
- 사용자는 초안을 삭제할 수 있다.
    - 작성자 본인만 초안을 수정할 수 있다.

### 게시물 좋아요 반응

- 모든 사용자는 게시물에 좋아요 반응을 보일 수 있다.
    - 삭제된 게시물은 좋아요 반응을 할 수 없다.
    - 중복된 게시물 좋아요 반응은 허용하지 않는다.
    - 개시물 좋아요 반응 생성시 작성자에게 알림이 전송되어야 한다.
- 사용자는 게시물 좋아요 반응을 취소할 수 있다.
    - 사용자는 삭제된 게시물에 남긴 게시물 좋아요 반응을 취소 할 수 없다.

## 용어사전

---

### 게시물

| 한글명    | 영문명         | 설명                                                          |
|--------|-------------|-------------------------------------------------------------|
| 게시물    | post        | 사용자가 작성한 글                                                  |
| 작성자    | author      | 게시물을 작성한 사용자                                                |
| 발행     | publish     | 사용자가 게시물의 작성하여 사용자들이 볼 수 있도록 등록하는 것                         |
| 수정     | modify      | 사용자가 게시물의 초안을 작성하여 잠시 저장해두는 것. 임시 저장된 게시물은 게시물 목록에 보여지지 않는다 |
| 삭제     | delete      | 사용자가 발행한 삭제하는 것. 삭제된 게시물은 게시물 목록에 보여지지 않는다                  |
| 제목     | title       | 게시물의 제목                                                     |
| 본문     | body        | 게시물에서 전달하고자 하는 내용                                           |
| 컨텐츠    | content     | 게시물의 실제 내용을 함께 일컫는 말(제목, 본문)                                |
| 게시물 상태 | post status | 게시물의 발행, 삭제 정보를 나타낸다                                        |
| 발행됨 상태 | published   | 게시물이 발행 완료된 상태                                              |
| 삭제됨 상태 | deleted     | 게시물이 삭제 완료된 상태                                              |

### 초안

| 한글명    | 영문명          | 설명                          |
|--------|--------------|-----------------------------|
| 초안     | draft        | 게시물을 발행하기 전 작성한 글           |
| 기안자    | drafter      | 게시물의 초안을 작성한 사용자            |
| 발행     | publish      | 사용자가 초안을 완성하여 게시물을 등록하는 것   |
| 수정     | modify       | 사용자가 초안의 내용을 변경하는 것         |
| 삭제     | delete       | 사용자가 작업중이던 초안을 삭제하는 것       |
| 제목     | title        | 초안의 제목                      |
| 본문 내용  | body         | 초안에서 전달하고자 하는 실제 내용         |
| 컨텐츠    | content      | 초안의 실제 내용을 함께 일컫는 말(제목, 본문) |
| 초안 상태  | draft status | 초안의 저장, 발행, 삭제 상태를 나타낸다     |
| 저장됨 상태 | saved        | 초안이 저장된 상태                  |
| 발행됨 상태 | published    | 초안이 게시물로 발행 완료된 상태          |
| 삭제됨 상태 | deleted      | 게시물이 삭제 완료된 상태              |

## 모델링

---

### Post

#### 속성

- ```Post``` 는 ```식별자(id)``` 를 가진다.
- ```Post``` 는 ```작성자(author)``` 정보를 가진다.
- ```Post``` 는 ```제목(title)```을 갖는다.
    - 제목은 반드시 1글자 이상이어야 한다.
    - 제목이 빈문자열이라면 ```Post``` 발행에 실패해야한다.
- ```Post``` 는 ```본문 내용(body)``` 을(를) 갖는다.
    - ```본문 내용(body)``` 는 비어있는 값일 수 있다.
- ```Post``` 는 ```상태값(status)``` 을(를) 가진다. 상태는 반드시 아래의 값중 1개의 값만을 가질 수 있다. 동시에 여러가지 값을 가질 수 없다.
    - ```PUBLISHED``` : 게시물이 발행되어진 상태. ```PUBLISHED``` 상태의 게시물은 게시물 목록에 노출되어진다.
    - ```DELETED``` : 게시물이 삭제되어진 상태. ```DELETED``` 상태의 게시물은 게시물 목록에 노출되지 않는다.

#### 행위

- ```Post``` 를 ```발행``` 할 수 있다. 발행된 ```Post``` 는 ```PUBLISHED``` 상태를 가진다.
- ```Post``` 를 ```수정``` 할 수 있다. 수정 완료된 ```Post``` 는 ```PUBLISHED``` 상태를 가진다.
    - ```작성자``` 본인만 ```Post``` 를 수정할 수 있다.
    - 삭제된 ```Post``` 는 수정할 수 없다.
- ```Post``` 를 ```삭제``` 할 수 있다. 삭제 상태의 ```Post``` 는 ```DELETED``` 상태를 가진다.
    - ```작성자``` 본인만 ```Post``` 를 삭제할 수 있다.
- ```Post``` 의 ```콘텐츠(제목, 본문 내용)``` 를 수정할 수 있다.
    - ```작성자``` 본인만 ```Post``` 를 수정할 수 있다.

### Draft

#### 속성

- ```Draft``` 는 ```식별자(id)``` 를 가진다.
- ```Draft``` 는 ```기안자(drafter)``` 정보를 가진다.
- ```Draft``` 는 ```제목(title)```을 갖는다.
    - 제목은 반드시 1글자 이상이어야 한다.
    - 제목이 빈문자열이라면 ```Draft``` 저장에 실패해야한다.
- ```Draft``` 는 ```본문 내용(body)``` 을(를) 갖는다.
    - ```본문 내용(body)``` 는 비어있는 값일 수 있다.
- ```Draft``` 는 ```상태값(status)``` 을(를) 가진다. 상태는 반드시 아래의 값중 1개의 값만을 가질 수 있다. 동시에 여러가지 값을 가질 수 없다.
    - ```SAVED``` : 초안이 저장되어진 상태.
    - ```PUBLISHED``` : 초안이 게시물로 발행 완료되어진 상태. ```PUBLISHED``` 상태의 초안은 삭제되어야 한다.
    - ```DELETED``` : 초안이 삭제되어진 상태.

#### 행위

- ```Draft``` 를 ```저장``` 할 수 있다. 저장된 ```Draft``` 는 ```SAVED``` 상태를 가진다.
- ```Draft``` 는 ```게시물``` 로 ```발행``` 될 수 있다. 발행된 ```Draft``` 는 ```PUBLISHED``` 상태를 가진다.
- ```Draft``` 를 ```수정``` 할 수 있다. 수정 완료된 ```Draft``` 는 ```SAVED``` 상태를 가진다.
    - ```기안자``` 본인만 ```Draft``` 를 수정할 수 있다.
    - 삭제된 ```Draft``` 는 수정할 수 없다.
- ```Draft``` 를 ```삭제``` 할 수 있다. 삭제 상태의 ```Draft``` 는 ```DELETED``` 상태를 가진다.
    - ```기안자``` 본인만 ```Draft``` 를 삭제할 수 있다.
- ```Draft``` 의 ```콘텐츠(제목, 본문 내용)``` 를 수정할 수 있다.
    - ```기안자``` 본인만 ```Draft``` 를 수정할 수 있다.
